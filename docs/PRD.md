# PRD: 개인 개발 블로그 (Notion CMS 기반)

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 개인 개발 블로그 |
| 목적 | Notion을 CMS로 활용한 개인 기술 블로그 |
| 대상 사용자 | 기술 블로그 독자 (개발자) |
| 배포 환경 | Vercel |

### 1.1 배경 및 목적

Notion에서 글을 작성하면 별도의 배포 없이 블로그에 자동으로 반영되는 개인 기술 블로그를 구축한다. 콘텐츠 관리를 Notion에 위임함으로써 글쓰기에만 집중할 수 있는 환경을 만드는 것이 목적이다.

### 1.2 CMS로 Notion을 선택한 이유

- 별도의 CMS 관리 화면 없이 익숙한 Notion 인터페이스에서 글 작성 가능
- Notion에서 글을 발행하면 블로그에 자동 반영 (실시간 연동)
- 데이터베이스 속성을 활용한 카테고리, 태그, 상태 관리 용이
- 무료 API 제공으로 추가 비용 없음

## 3. Notion 데이터베이스 구조

| 속성명 | 타입 | 설명 | 비고 |
|--------|------|------|------|
| Title | `title` | 글 제목 | 필수 |
| Category | `select` | 카테고리 | 예: Frontend, Backend, DevOps |
| Tags | `multi_select` | 태그 | 예: React, TypeScript, Docker |
| Published | `date` | 발행일 | |
| Status | `select` | 상태 | `초안` / `발행됨` |
| Content | Page Content | 본문 | Notion 페이지 블록 사용 |

> **조회 조건**: `Status === '발행됨'` 인 항목만 블로그에 노출

## 4. 주요 기능

### 4.1 Notion 데이터베이스 연동

- Notion API를 통해 데이터베이스에서 글 목록 조회
- 상태가 `발행됨`인 글만 필터링하여 노출
- 글 본문은 Notion 페이지 블록을 파싱하여 렌더링

### 4.2 글 목록 페이지

- 최근 발행된 글을 카드 형태로 목록 표시
- 제목, 카테고리, 태그, 발행일 표시
- 페이지네이션 또는 무한 스크롤 (MVP 이후 검토)

### 4.3 글 상세 페이지

- 개별 글의 본문 렌더링 (Notion 블록 → HTML/JSX 변환)
- 지원 블록 타입: 단락, 제목(H1~H3), 코드, 이미지, 인용, 목록
- 메타 정보(카테고리, 태그, 발행일) 표시

### 4.4 카테고리별 필터링

- 카테고리 선택 시 해당 카테고리의 글만 필터링
- URL 파라미터 기반으로 카테고리 상태 관리 (`?category=Frontend`)

### 4.5 검색 기능

- 글 제목 기준 클라이언트 사이드 검색
- 검색어 입력 시 실시간 필터링

### 4.6 반응형 디자인

- 모바일 / 태블릿 / 데스크탑 레이아웃 지원
- Tailwind CSS 반응형 유틸리티 사용

## 5. 화면 구성

### 5.1 홈 (`/`)

```
┌─────────────────────────────────┐
│  [로고/블로그명]       [검색]   │  ← Header
├─────────────────────────────────┤
│  카테고리 필터 탭               │
│  [전체] [Frontend] [Backend]... │
├─────────────────────────────────┤
│  ┌────────┐ ┌────────┐          │
│  │ 글 카드 │ │ 글 카드 │  ...   │  ← 글 목록 Grid
│  │ 제목   │ │ 제목   │          │
│  │ 카테고리│ │ 카테고리│         │
│  │ 날짜   │ │ 날짜   │          │
│  └────────┘ └────────┘          │
└─────────────────────────────────┘
```

### 5.2 글 상세 (`/posts/[slug]`)

```
┌─────────────────────────────────┐
│  [로고/블로그명]       [검색]   │  ← Header
├─────────────────────────────────┤
│  < 목록으로 돌아가기            │
│                                 │
│  # 글 제목                      │
│  [카테고리] [태그1] [태그2]     │
│  발행일: 2025-01-01             │
│  ─────────────────────────      │
│                                 │
│  글 본문 내용...                │
│                                 │
└─────────────────────────────────┘
```

### 5.3 카테고리 (`/?category=[name]`)

- 홈 화면에서 카테고리 필터가 적용된 상태와 동일
- 선택된 카테고리 탭 하이라이트 표시

## 7. 구현 단계

### 1단계: 환경 설정

- Next.js 15 프로젝트 초기화 (App Router)
- TypeScript, Tailwind CSS, shadcn/ui 설치
- `@notionhq/client` 패키지 설치
- `.env.local`에 Notion API 키 및 데이터베이스 ID 설정

```env
NOTION_API_KEY=secret_xxxx
NOTION_DATABASE_ID=xxxx
```

### 2단계: Notion 데이터베이스 구성

- Notion에서 블로그용 데이터베이스 생성
- 3.절의 속성 구조에 따라 속성 추가
- Integration 연결 후 API 키 발급

### 3단계: 글 목록 페이지 구현

- Notion API 클라이언트 유틸리티 작성 (`src/lib/notion.ts`)
- 글 목록 조회 함수 구현 (필터: `Status === '발행됨'`)
- 홈 페이지 컴포넌트 작성 (카드 그리드, 카테고리 필터)
- ISR(Incremental Static Regeneration) 적용 (`revalidate` 설정)

### 4단계: 글 상세 페이지 구현

- 글 상세 조회 함수 구현 (페이지 블록 파싱)
- 블록 타입별 렌더링 컴포넌트 작성
- 동적 라우트 생성 (`generateStaticParams`)

### 5단계: 스타일링 및 최적화

- shadcn/ui 컴포넌트 적용 (Card, Badge, Input 등)
- 반응형 레이아웃 최종 점검
- Vercel 배포 및 환경변수 설정
- Lighthouse 성능 점검
